Frame2 Eclipse Changelog

Version 1.4.5

  • Update with latest Frame2 (version 1.3.1) libraries.

Version 1.4.3

  • Add Frame2 nature when creating new project, or by right clicking a project
  • Added project decorator - tiny Frame2 logo on projects with the Frame2 nature
  • Added Frame2 Builder - check for errors in config on projects with Frame2 nature
  • Right click frame2-config.xml to access Frame2 wizards
  • Added logging capability - plugin errors are logged to the default Eclipse error log
  • Properly set window titles for all wizards
  • frame2-config.xml stays in sync with workbench when modified by a wizard

Version 1.3.3

  • Updated Getting Started guide with new icons
  • Grouped Getting Started Guides in Help
  • When creating an Event as "Existing Class", look up classes in the project that extend CommonsValidatorEvent and JaxbEventBase as well as the ones defined in the config.
  • Support setting "resolveAs" when creating an event from a class that extends JaxbEventBase
  • When creating a Global Forward of type Responder, look up all implementors of Responder interface for user selection
  • Refactor so that the Web Services plugin can access wizards, constants, etc.
  • Released alongside the Web Services plugin

Version 1.3.1

  • Fix possible NullPointerException in object model

Version 1.3

  • Update to work with Frame2 version 1.3
  • Update to comply to Eclipse 3.3 API standards
  • Cleanup code - eliminate warnings and FindBugs issues
  • Added What is Frame2 document
  • Added license information document
  • Added tag attribute mappings document
  • Updated Getting Started Guide
  • Added Frame2 JavaDoc to help
  • Added Getting Started with Templates document
  • Added third party license document
  • Modified model to work with version 1.2 of Frame2 DTD
  • Heavily refactored domain model to be smaller and faster
